3. \# 供电自动化拓扑链路架构图文字说明文档 ## 一、文档说明 本文档基于供电自动化拓扑的设备类型与输出链路，以文字描述形式呈现专业输出链路、跨专业输出链路的层级结构、设备连接逻辑及关键节点关系，替代可视化架构图，便于快速理解链路整体架构。 --- ## 二、专业输出链路架构说明（供电专业内部） ### （一）架构整体定位 专业输出链路为“纵向分层串联架构”，按“高压层→低压层→末端配电层”划分3个核心层级，仅包含供电专业设备，各层级通过关键转换设备（变压器、电源）实现衔接，无跨专业设备接入。 ### （二）分层架构与连接逻辑 #### 1. 高压层架构（链路起点） - **层级定位**：负责外部高压电源接入与初步分配，为低压层提供高压输入。 - **设备组成**：高压进线柜（1台，链路起点）→ 高压出线柜（1-N台，按低压回路数量配置）→ 高压母联柜（1台，可选，仅双回路高压场景配置）→ 变压器（1-N台，与高压出线柜一一对应）。 - **连接逻辑**：  1. 外部高压电源（如10kV）首先接入高压进线柜；  2. 高压进线柜通过电缆连接至高压出线柜，1台高压出线柜对应1台变压器；  3. 若为双回路高压，2组“高压进线柜→高压出线柜”通过高压母联柜连接，实现回路切换；  4. 高压出线柜最终连接至变压器，完成高压层到低压层的过渡（变压器为高压层终点、低压层起点）。 #### 2. 低压层架构（链路中间层） - **层级定位**：将变压器转换后的低压电（如380V/220V）分配至末端配电设备，含不间断供电保障模块。 - **设备组成**：低压进线柜（1台，与变压器一一对应）→ 低压出线柜（1-N台，按末端配电需求配置）→ 低压母联柜（1台，可选，双回路低压场景配置）→ UPS主机/开关电源/直流屏（1-N台，按供电可靠性需求配置）。 - **连接逻辑**：  1. 变压器输出的低压电接入低压进线柜，完成“高压→低压”转换后的首次低压分配；  2. 低压进线柜连接至低压出线柜，1台低压出线柜对应1路末端供电回路；  3. 双回路低压场景下，2组“低压进线柜→低压出线柜”通过低压母联柜连接，实现低压回路备份；  4. 低压出线柜分别连接至UPS主机（需不间断供电场景，如数据中心）、开关电源（直流供电场景，如基站）或直流屏（应急供电场景），为末端配电层提供稳定低压输入。 #### 3. 末端配电层架构（链路终点） - **层级定位**：实现低压电的最终分配，为末端设备提供直接供电接口。 - **设备组成**：列头柜（1台，区域级分配）→ PDU（1-N台，设备级接口）。 - **连接逻辑**：  1. UPS主机/开关电源/直流屏输出的低压电，首先接入列头柜，按机房区域（如机柜列）进行二次分配；  2. 1台列头柜对应1列或多个机柜，通过电缆连接至各机柜内的PDU；  3. PDU作为末端配电终点，直接为机柜内的设备（如服务器、网络设备）提供电源接口，完成专业输出链路的全流程。 #### 4. 无高压场景架构（简化版） - **适用场景**：无高压供电的小型站点（如部分基站、接入机房）。 - **架构逻辑**：省略高压层，直接从低压进线柜开始，按“低压进线柜→低压出线柜→开关电源→列头柜→PDU”路径构建链路，其他连接逻辑与含高压场景的低压层、末端配电层一致。 --- ## 三、跨专业输出链路架构说明（多专业协同） ### （一）架构整体定位 跨专业输出链路为“供电专业为核心+横向延伸”架构，以专业输出链路为基础，横向连接IT、空调、动环等专业设备，形成“供电→用电→监控”的完整闭环链路。 ### （二）分场景架构与连接逻辑 #### 1. 供电→IT专业链路（最核心跨专业场景） - **基础链路**：复用专业输出链路的“高压层→低压层→末端配电层”全路径（高压进线柜→...→PDU）。 - **跨专业延伸**：PDU（专业输出链路终点）→ IT专业设备（服务器/网络设备/存储设备）。 - **关键逻辑**：  1. 1台PDU对应多个IT设备，按设备功率需求分配电源接口；  2. IT设备为链路最终用电终端，无后续延伸设备；  3. 链路核心价值：通过供电链路保障IT业务设备运行，支撑核心业务（如数据处理、网络传输）。 #### 2. 供电→空调专业链路（环境保障场景） - **基础链路**：复用专业输出链路的“高压层→低压层”路径（高压进线柜→...→低压出线柜），新增“空调配电柜”（供电专业设备，衔接低压层与空调设备）。 - **跨专业延伸**：低压出线柜→空调配电柜→空调专业设备（精密空调/新风空调）。 - **关键逻辑**：  1. 空调配电柜为专属配电设备，独立于末端配电层的列头柜，避免与IT设备供电冲突；  2. 1台空调配电柜对应1-N台空调设备，按空调功率配置供电回路；  3. 链路核心价值：为空调设备供电，控制机房温湿度（如精密空调维持22±2℃），保障供电、IT设备稳定运行。 #### 3. 供电→动环专业链路（监控保障场景） - **基础链路**：复用专业输出链路的“高压层/低压层”部分路径（高压进线柜/低压出线柜→开关电源）。 - **跨专业延伸**：开关电源→动环专业设备（动环监控主机→电流/电压传感器/温湿度传感器）。 - **关键逻辑**：  1. 动环监控主机为核心控制节点，通过开关电源获取稳定供电；  2. 传感器分两类：     - 电流/电压传感器：直接连接至高压进线柜、低压出线柜等供电设备，采集供电参数；     - 温湿度传感器：部署于机房空间或关键设备旁，采集环境参数；  3. 所有传感器数据汇总至动环监控主机，实现“供电状态+环境状态”统一监控，异常时触发告警（如电压过高、温度超标）。 #### 4. 供电→IT+动环专业链路（多专业协同场景） - **基础链路**：复用“供电→IT专业链路”全路径（高压进线柜→...→PDU→服务器）。 - **跨专业延伸**：  - 分支1：PDU→服务器（IT专业）；  - 分支2：开关电源→动环监控主机→传感器（动环专业，传感器同时监测供电设备、服务器、机房环境）。 - **关键逻辑**：  1. 链路为“主干+分支”结构，主干为专业输出链路，分支1连接IT设备、分支2连接动环设备；  2. 动环传感器实现“全链路覆盖”：既监测高压进线柜、低压出线柜等供电设备，也监测服务器运行状态（如服务器温度）、机房环境；  3. 链路核心价值：形成闭环管理，当供电异常（如电压低）时，动环监控主机可快速定位故障点，并关联IT设备运行状态（如服务器是否离线），提升运维效率。 --- ## 四、架构共性关键规则 1. **层级衔接规则**：专业输出链路中，高压层与低压层通过“变压器”衔接，低压层与末端配电层通过“UPS/开关电源”衔接；跨专业链路中，供电专业与其他专业通过“专属配电设备”（如PDU、空调配电柜）衔接，避免直接跨专业连接。 2. **冗余配置规则**：双回路场景（高压/低压）均配置母联柜，实现回路备份；关键设备（如UPS、动环监控主机）均配置独立供电回路，避免单点故障影响全链路。 3. **场景适配规则**：不同站点（数据中心、基站、核心机楼）的链路架构，均基于“基础层级+可选模块”调整（如基站无高压层、数据中心需UPS模块），确保架构灵活性。 要不要我针对其中某类链路（如供电→IT专业链路），进一步整理一份**设备连接检查清单**？清单会明确每个连接节点的检查项（如接线正确性、参数匹配性），方便运维人员进行链路验证。





\# 设备类型分类及输出链路清单 ## 一、设备类型分类 根据供电自动化拓扑配置及输出链路需求，设备类型按专业属性及功能角色划分为以下类别，覆盖高压、低压、末端配电全链路。 | 设备类别 | 具体设备类型 | 所属专业 | 核心功能 | | ---- | ---- | ---- | ---- | | **高压侧设备** | 高压柜、高压进线柜、高压出线柜、高压母联柜、变压器 | 供电专业 | 实现高压电输入、分配、转换，为低压系统提供电源 | | **低压侧设备** | 低压柜、低压进线柜、低压出线柜、低压母联柜、UPS主机、UPS电池组 | 供电专业 | 将高压转换后的低压电分配至末端，UPS保障不间断供电 | | **电源设备** | 开关电源、直流屏、列头柜、PDU（电源分配单元） | 供电专业 | 实现低压电的二次分配，为末端设备提供稳定电源接口 | | **末端用电设备** | 服务器、网络设备（交换机、路由器）、存储设备、空调设备 | IT/空调专业 | 消耗电能实现业务承载（IT设备）或环境控制（空调设备） | | **监控设备** | 动环监控主机、传感器（电流/电压传感器、温湿度传感器） | 动环专业 | 采集设备运行数据，实现供电链路状态监控与告警 | --- ## 二、专业输出链路 专业输出链路指同一供电专业内部，从高压到低压再到末端配电的完整供电链路，仅包含供电专业所属设备，链路节点按配电层级依次串联。 ### （一）高压层→低压层链路 1. **链路路径**：高压进线柜 → 高压出线柜 → 高压母联柜（可选，双回路场景） → 变压器 → 低压进线柜 2. **关键特性**    - 链路起点为外部高压电源接入点，终点为低压进线柜（低压系统入口）。    - 仅包含供电专业高压侧、低压侧核心设备，无跨专业设备接入。    - 支持双回路高压供电场景，通过高压母联柜实现回路切换。 ### （二）低压层→末端配电层链路 1. **链路路径**：低压进线柜 → 低压出线柜 → 低压母联柜（可选） → UPS主机（可选） → 开关电源/直流屏 → 列头柜 → PDU 2. **关键特性**    - 链路起点为低压进线柜，终点为PDU（末端设备电源接口）。    - 包含UPS、开关电源等供电专业关键设备，实现电源稳压、不间断供电功能。    - 支持按“分路顺序号”划分多条并行链路，每条链路对应独立电源分配路径。 --- ## 三、跨专业输出链路 跨专业输出链路指以供电专业为核心，延伸至IT、空调、动环等其他专业的完整供电链路，覆盖“供电→用电→监控”全流程，包含多专业设备协同。 ### （一）供电→IT专业链路 1. **链路路径**：高压进线柜 → 变压器 → 低压出线柜 → UPS主机 → 列头柜 → PDU → 服务器/网络设备/存储设备 2. **跨专业节点**：服务器、网络设备、存储设备（IT专业） 3. **关键特性**    - 核心作用：为IT专业业务设备提供稳定供电，保障业务系统运行。    - 链路依赖：需通过“动环监控设备ID”关联供电设备与IT设备，实现供电状态对IT设备的影响监控。 ### （二）供电→空调专业链路 1. **链路路径**：高压进线柜 → 变压器 → 低压出线柜 → 开关电源 → 空调配电柜 → 空调设备（精密空调、新风空调） 2. **跨专业节点**：空调设备（空调专业） 3. **关键特性**    - 核心作用：为空调专业环境控制设备供电，保障机房温湿度稳定。    - 链路要求：需支持大功率空调设备的供电需求，链路设备需满足高负载要求。 ### （三）供电→动环专业链路 1. **链路路径**：高压进线柜/低压出线柜 → 开关电源 → 动环监控主机 → 传感器（电流/电压/温湿度） 2. **跨专业节点**：动环监控主机、传感器（动环专业） 3. **关键特性**    - 核心作用：为动环专业监控设备供电，实现供电链路运行数据采集与告警。    - 数据关联：通过动环监控设备采集供电设备的电流、电压等参数，实现跨专业数据互通。 ### （四）多专业协同链路 1. **链路路径**：高压进线柜 → 变压器 → 低压出线柜 → UPS主机 → 列头柜 → PDU → 服务器 + 动环传感器 → 动环监控主机 2. **跨专业节点**：服务器（IT专业）、动环监控主机/传感器（动环专业） 3. **关键特性**    - 核心作用：整合供电、IT、动环三个专业，实现“供电状态→IT设备运行→监控告警”的全链路协同。    - 链路价值：通过跨专业设备关联，可快速定位“供电异常→IT设备故障”的根因，提升运维效率
