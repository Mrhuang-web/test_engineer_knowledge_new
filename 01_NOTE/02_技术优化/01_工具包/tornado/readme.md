tornado服务器理解
    1、可以扩展到数万个开放连接【比如：用户的浏览器和服务器之间建立的网络通道】 
    2、使其成为 长轮询、WebSockets能和其他需要与每个用户建立持久连接的应用

传统服务器与tornado服务器区别
    传统服务器
        1、传统服务器（如Apache、Tomcat的默认模式）
        2、它们通常采用“一个连接一个线程”的模型【如果后端处理很慢，那么线程就会一直等待，新进来的又得占新的线程】
        3、每当一个用户访问服务器，服务器就分配一个专门的线程去处理这个用户的请求
    Tornado服务器
        1、它采用了异步非阻塞和事件循环的模型。
        2、可以把它想象成一个极其高效的单人接待员，
