# 补充综资知识【重点】

```
综资作用：
	有效促进动环专业、能耗、用电情况、供电情况、能流输出情况、
	设备补充协议、拓扑图、跨专业输出分路影响等

综资获取方式：
    目前综资已经将动环专业的数据保存在共享平上。
    共享平台提供API接口可调用获取综资数据
    通过集中化4A平台，登录共享平台数据订阅方式，可申请数据订阅
    通过“订单编码”作为共享平台获取数据API授权

综资更新频次
	1.数据更新频率【每4天更新1次数据】
```

```
站点动环属性
	站点名称：
		(关联)填写综合资源系统站点名称。
		关联到【空间资源】-站点-资源标识
	动环监控站点名称：
		标题：动环站点名称
		1）如果名称能匹配到动环平台站点，则增加超链接，
		并跳转都【综合-实时性能】对应站点；
		2）如果不能匹配，则红色字体展示。
		鼠标移动时，文案提示：无法匹配总部动环站点。
	动环监控站点ID：
		动环监控系统中的站点ID
		默认不展示



动环与综资对象匹配【重点】
	综资-园区、站点理解：
		综资的数据中心是指园区：对应动环的站点（类型为：数据中心）
		综资的站点：对应动环 核心机楼+数据中心的楼栋
			核心机楼：核心站点、核心站点（配套），包含数据中心楼栋
			传输节点：骨干站点、汇聚站点
			通信基站：接入站点、用户站点
	
	匹配的内容：
		即站点动环属性和机房动环属性，以及设备综资：
			关联到动环时需要匹配的【以下是匹配方式】
		站点动环属性：
			是通过zh_label:关联到【空间资源】-站点-资源标识(int_id)
			然后就是对应的资源表示再与映射表之间的匹配
		
	
	匹配方式1：
		通过综资提供的映射关系【es的索引模型】：
			园区映射关系【ods_zz_irms_dc_map】
				通过综资RM_AREA_dc模型资源标识（int_id）
					在这里匹配zg_id，拿到动环dh_id或dh_name
				通过综资RM_AREA_dc模型位置点名称（zh_label）
					在这里匹配zg_name，拿到动环dh_id或dh_name
			站点映射关系【ods_zz_irms_site_map】
				通过综资RM_AREA_site模型资源标识（int_id）
					在这里匹配zg_id，拿到动环dh_id或dh_name
				通过综资RM_AREA_site模型位置点名称（zh_label）
					在这里匹配zg_name，拿到动环dh_id或dh_name
			机房映射关系【ods_zz_irms_rom_map】
				通过综资RM_AREA_ROOM模型资源标识（int_id）
					在这里匹配zg_id，拿到动环dh_id或dh_name
				通过综资RM_AREA_ROOM模型位置点名称（zh_label）
					在这里匹配zg_name，拿到动环dh_id或dh_name
	匹配方式2：
		1、先进行省市级别的匹配【区域级别】
			所有空间表+动环属性表：
				会先于动环t_cfg_precinct进行匹配省市
				即t_cfg_precinct的area_code
					[对应 区域信息（区县及以上）字典表]
					去对应综资的
		2-1、区域级别匹配成功则进行数据中心匹配【数据中心站点级别】
			数据中心（空间）：
				利用里面动环里面的数据中心站点名称
				与数据中心（空间）zh_label进行匹配
		2-2、区域级别匹配成功则进行站点匹配【其余站点级别】
			站点（空间）/站点动环属性：
				利用里面动环里面的站点名称
				与站点（空间）/站点动环属性 zh_label进行匹配
		2-3、区域级别匹配成功则进行楼栋匹配【数据中心楼栋级别】
			站点（空间）/站点动环属性：
				利用里面动环里面的站点名称【站点（空间还涉及related_dc）】
				与站点（空间）/站点动环属性 zh_label进行匹配
		3、区域级别和站点级别都匹配到后进行机房匹配【机房级别】
			机房（空间）/机房动环属性
				利用里面动环里面的机房名称或所属站点
				与机房（空间）/机房动环属性的zh_label进行匹配
				或所属站点（related_site/power_related_site_name）进行匹配
```
