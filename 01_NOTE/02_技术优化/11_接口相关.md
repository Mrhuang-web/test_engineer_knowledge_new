# 01接口测试方法即工具

```
方式1：
	curl：
		基础语法：【curl 要测试的接口】
	案例【post类型】：
		curl -d '{"name":"yupi"}' -H "Content-type:application/json" 测试链接
			-d：要发送的数据
			-H：指定请求头
			
	快捷方式：
		直接f12，然后右键接口，以curl复制即可
	注意点：
		使用curl的话，要复制bash的进行粘贴
		如果使用它cmd的话，没办法调用，只能在电脑的cmd里面进行调用，无法倍工具解析
	
	
	
方式2：
	apifox：
		支持多种个接口类型导入
		一键调试接口，本地后台服务不启动，可以mock快速生成模拟数据
		
		
		
方式3：
	swagger：
		
		
方式4：
	Knife4j
```



# 02接口响应导出JSON

```
方法1：
	f12、打开接口的响应 -- response或preview、右键复制粘贴到自己的json文件上
	
方法2：
	f12、右键对应接口、复制curl(cmd)
	复制到cmd里面，并且在最后面添加 > filename.json    [可以先编辑好，并且filename这里的可以改成对应路径，即可生成成功]
	
方法3：
	使用python的requests、结合json库，将response转化为json、并用with open写入即可
```
